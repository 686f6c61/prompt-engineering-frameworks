El poder de los frameworks de prompting

Prompt Agent utiliza un sistema avanzado de análisis para seleccionar el framework ideal entre nuestra colección de 47 marcos especializados. Cada framework está diseñado para maximizar la efectividad de tus interacciones con modelos de lenguaje como OpenAI, Claude, Llama, Deepseek y otros LLMs.

¿Por qué necesitamos buenos prompts?

Los modelos de lenguaje (LLM) dependen enormemente de la calidad y estructura de los prompts que reciben. Un prompt bien estructurado puede:

  • Reducir la ambigüedad - Permitiendo respuestas más precisas y relevantes
  • Mejorar la consistencia - Estructurando la información de manera lógica y coherente
  • Optimizar tokens - Ahorrando costos al reducir la necesidad de iteraciones y correcciones

Beneficios para Bolt y Lovable

Cuando utilizas Prompt Agent para generar especificaciones para plataformas como Bolt y Lovable, obtienes:

  • Ahorro de tokens - Las instrucciones claras y estructuradas reducen el procesamiento necesario
  • Mayor precisión - La estructura correcta ayuda a Bolt/Lovable a entender exactamente qué desarrollar
  • Desarrollo más rápido - Especificaciones claras reducen el tiempo de implementación
  • Menos iteraciones - Prompts bien estructurados reducen la necesidad de correcciones y ajustes

Proceso:

  1. Describes tu objetivo
  2. Nuestro sistema analiza tu necesidad
  3. Selecciona el framework más apropiado
  4. Proporciona una explicación detallada
  5. Muestra un ejemplo práctico de uso

Frameworks disponibles

1. RTF (Role, Task, Format)

Framework para estructurar prompts definiendo el rol del AI, la tarea específica y el formato deseado de respuesta.

Ideal para: Generación de contenido estructurado y tareas específicas.

2. PARA (Purpose, Action, Result, Assessment)

Define propósito, acciones necesarias, resultados esperados y métodos de evaluación.

Ideal para: Planificación y evaluación de proyectos.

3. SMART (Specific, Measurable, Achievable, Relevant, Time-bound)

Marco para establecer objetivos claros, medibles y alcanzables en un tiempo determinado.

Ideal para: Definición de objetivos y metas.

4. ERQ (Experience, Requirements, Qualifiers)

Framework para definir experiencia necesaria, requisitos específicos y calificadores del proyecto.

Ideal para: Definición de perfiles y requisitos.

5. CODE (Context, Objective, Details, Examples)

Estructura para proporcionar contexto completo, objetivos claros, detalles específicos y ejemplos ilustrativos.

Ideal para: Documentación técnica y guías.

6. PROS (Perspective, Requirements, Outcome, Scope)

Marco para definir perspectivas, requisitos, resultados esperados y alcance del proyecto.

Ideal para: Planificación de proyectos.

7. TEAM (Task, Environment, Approach, Metrics)

Framework para definir tareas, entorno de trabajo, enfoque metodológico y métricas de éxito.

Ideal para: Trabajo en equipo y colaboración.

8. IDEA (Identify, Define, Execute, Assess)

Marco para identificar problemas, definir soluciones, ejecutar planes y evaluar resultados.

Ideal para: Resolución de problemas.

9. CARE (Context, Action, Result, Example)

Proporciona contexto completo, define acciones, especifica resultados esperados e incluye ejemplos.

Ideal para: Instrucciones detalladas y aprendizaje.

10. RISE (Role, Input, Steps, Expectation)

Define roles, entradas necesarias, pasos a seguir y expectativas del resultado.

Ideal para: Procesos y flujos de trabajo.

11. LOGIC (Layout, Objective, Guidelines, Implementation, Criteria)

Estructura para desarrollar soluciones con un enfoque lógico y sistemático.

Ideal para: Desarrollo de sistemas y procesos.

12. SCOPE (Situation, Core Need, Obstacles, Plan, Evaluation)

Marco para definir el alcance del proyecto considerando obstáculos y evaluación.

Ideal para: Definición y gestión de alcance.

13. FOCUS (Frame, Objective, Constraints, Understanding, Solution)

Framework para enmarcar problemas y desarrollar soluciones considerando limitaciones.

Ideal para: Resolución estructurada de problemas.

14. CLARITY (Context, Limitations, Approach, Requirements, Implementation, Timeline, Yield)

Framework para planificar y ejecutar proyectos con un enfoque claro en limitaciones y resultados.

Ideal para: Gestión de proyectos complejos y migraciones.

15. EXPERT (Expertise, Context, Purpose, Execution, Results, Testing)

Marco completo para proyectos que requieren experiencia especializada y pruebas.

Ideal para: Proyectos técnicos especializados.

16. GUIDE (Goal, User, Implementation, Delivery, Evaluation)

Framework centrado en el usuario para desarrollar soluciones efectivas y medibles.

Ideal para: Desarrollo de productos y servicios.

17. PATH (Purpose, Approach, Target, Horizon)

Marco para definir la dirección estratégica y los objetivos a largo plazo.

Ideal para: Planificación estratégica.

18. LEARN (Level, Experience, Approach, Resources, Next steps)

Estructura para crear planes de aprendizaje y desarrollo personalizados.

Ideal para: Planes de formación y desarrollo.

19. SOLVE (Situation, Options, Limitations, Verification, Execution)

Marco sistemático para abordar y resolver problemas complejos.

Ideal para: Resolución de problemas complejos.

20. PRIME (Problem, Research, Implementation, Monitoring, Evaluation)

Framework para proyectos basados en investigación y mejora continua.

Ideal para: Proyectos de investigación y desarrollo.

21. ADAPT (Analysis, Design, Approach, Progress, Testing)

Marco flexible para adaptar soluciones a contextos cambiantes.

Ideal para: Proyectos adaptativos y ágiles.

22. BUILD (Baseline, Understanding, Implementation, Learning, Delivery)

Framework para construir soluciones desde cero con enfoque iterativo.

Ideal para: Desarrollo de nuevos productos o servicios.

23. CRAFT (Context, Requirements, Approach, Features, Testing)

Marco para desarrollar soluciones personalizadas con atención al detalle.

Ideal para: Desarrollo de soluciones a medida.

24. SCALE (Strategy, Capabilities, Action, Learning, Evolution)

Framework para escalar proyectos y soluciones de manera sostenible.

Ideal para: Escalamiento de proyectos y operaciones.

25. THINK (Topic, History, Insights, Next steps, Knowledge)

Marco para análisis profundo y toma de decisiones informada.

Ideal para: Análisis estratégico y toma de decisiones.

26. GROW (Goal, Reality, Options, Way Forward)

Marco para coaching y desarrollo personal enfocado en objetivos y acciones.

Ideal para: Coaching y desarrollo personal.

27. QUEST (Question, Understanding, Exploration, Solution, Testing)

Framework para exploración y resolución de problemas complejos.

Ideal para: Investigación y resolución de problemas.

28. DRIVE (Direction, Resources, Implementation, Validation, Evolution)

Marco para impulsar proyectos con enfoque en recursos y validación continua.

Ideal para: Gestión de proyectos innovadores.

29. SHAPE (Situation, History, Analysis, Plan, Execution)

Framework para dar forma a proyectos considerando contexto histórico y análisis.

Ideal para: Transformación organizacional.

30. REACH (Requirements, Evaluation, Approach, Completion, Handover)

Marco para llevar proyectos desde requisitos hasta entrega final.

Ideal para: Gestión de proyectos end-to-end.

31. BLEND (Baseline, Learning, Evolution, Navigation, Delivery)

Framework para integrar diferentes enfoques y metodologías en un proyecto.

Ideal para: Proyectos híbridos y metodologías mixtas.

32. SPARK (Strategy, Planning, Action, Results, Knowledge)

Marco para iniciar y catalizar proyectos innovadores con resultados medibles.

Ideal para: Iniciativas de innovación.

33. PULSE (Purpose, Understanding, Learning, Strategy, Evaluation)

Framework para monitorear y mantener la vitalidad de proyectos continuos.

Ideal para: Monitoreo y mejora continua.

34. FAST (Focus, Audience, Scope, Tone)

Framework para crear documentación técnica clara y efectiva según la audiencia objetivo.

Ideal para: Documentación técnica y guías de usuario.

35. T-A-G (Tarea, Acción, Meta)

Framework Tarea-Acción-Meta para prompts orientados a objetivos.

Perfecto para proyectos con objetivos medibles y acciones concretas

36. B-A-B (Antes, Después, Puente)

Framework Antes-Después-Puente para escenarios de transformación.

Excelente para mostrar cambios y mejoras con un plan claro

37. P-E-A-S (Propósito, Resultado, Audiencia, Estilo)

Framework Propósito-Resultado-Audiencia-Estilo para comunicación efectiva.

Perfecto para estrategias de comunicación y marketing

38. S-T-A-R (Situación, Tarea, Acción, Resultado)

Framework Situación-Tarea-Acción-Resultado para casos de éxito.

Excelente para documentar logros y experiencias

39. Q-C-Q-A (Pregunta, Contexto, Calificación, Respuesta)

Framework Pregunta-Contexto-Calificación-Respuesta para consultas estructuradas.

Ideal para resolver problemas complejos paso a paso

40. A-I-D-A (Atención, Interés, Deseo, Acción)

Framework Atención-Interés-Deseo-Acción para marketing persuasivo.

Perfecto para crear contenido persuasivo y campañas

41. L-E-A-P (Nivel, Expectativas, Enfoque, Parámetros)

Framework Nivel-Expectativas-Enfoque-Parámetros para planificación estratégica.

Óptimo para planificación estratégica

42. S-P-I-N (Situación, Problema, Implicación, Necesidad)

Framework Situación-Problema-Implicación-Necesidad para ventas consultivas.

Perfecto para ventas y negociación

43. D-E-S-I-G-N (Define, Explore, Scope, Ideate, Guide, Narrow)

Framework Define-Explore-Scope-Ideate-Guide-Narrow para diseño.

Ideal para procesos de diseño y desarrollo

44. V-I-S-I-O-N (Visualize, Identify, Structure, Implement, Optimize, Navigate)

Framework Visualize-Identify-Structure-Implement-Optimize-Navigate para planificación estratégica.

Perfecto para planificación estratégica

45. I-M-P-A-C-T (Intent, Message, Purpose, Audience, Channel, Timing)

Framework Intent-Message-Purpose-Audience-Channel-Timing.

Óptimo para estrategias de comunicación

46. M-A-S-T-E-R (Mission, Approach, Strategy, Tactics, Execution, Review)

Framework Mission-Approach-Strategy-Tactics-Execution-Review.

Ideal para gestión de proyectos

47. P-O-W-E-R (Problem, Outcome, Why, Execution, Resources)

Framework Problem-Outcome-Why-Execution-Resources.

Perfecto para resolución de problemas
